XADisk enables transactional access to file-systems, supporting both local and XA transactions.

 

Features

  • Enables transactional access to file-systems. Right now, supports only local file-system and this software needs to be running on the same JVM as the clients (this would not be a constraint in an upcoming enhancement).
  • JCA compliant, so can plug into any J2EE server implementation. J2EE applications will view it as a JCA Resource Adapter. 
  • Standalone Java applications can also be clients,
  • Enables MDBs to listen for changes for the files they are interested in. Such file state changes are abstracted as "messages" and delivered to the interested MDBs. 
  • Fulfills other expectations from transactional system : deadlock detection, concurrency control, crash-recovery, transaction timing-out etc.
  • Supports both local and XA transactions.

 

Testing

      Has been tested using automated test-cases in complex scenarios, including, 

  • forced JVM crashes followed by Crash Recovery,
  • Open/XA correctness,
  • concurrent transactions on common files/directories,
  • file-tree-structure as well as file-content level result comparisons.
