XADisk provides transactional access to the existing file-systems. It can be deployed over any Java 5/above JVM, including plain Java programs, Web Servers (like Tomcat), JavaEE Servers (like JBoss) etc.

It can also be deployed as a JCA (1.5) Resource Adapter in JavaEE Servers.

 

Features

  • Supports both XA and plain local transactions.
  • Is JCA 1.5 compliant, so can plug into any JavaEE 1.5+ Server.
  • JavaEE applications can interact with XADisk as they can with any other JCA Resource Adapter.
  • Standalone Java applications/Web Applications can also be clients to XADisk.
  • Enables MDBs to listen for changes in files/directories they are interested in.
  • Fulfills other expectations from a transactional system including deadlock detection, concurrency control, crash-recovery and transaction timing-out.
  • Can be invoked from Java/JavaEE applications running on the same or remote JVMs/machines.
  • Implemented using Java NIO and claims to support most of the file systems including FAT, NTFS, EXT etc.
