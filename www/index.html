Project Description:

"Summary: XADisk enables transactional access to file-systems, supporting both local and XA transactions."

"Here is a brief summary of the current implementation:
____________________________________________

-Enables transactional access to file-systems. Right now, supports only local file-system and this software needs to be running on the same JVM as the clients (can overcome it using network communication - i guess the implementation must be simple).
-JCA compliant, so can plug into any J2EE server implementation. J2EE applications will view it as a JCA Resource Adapter.
-Standalone Java applications can also be clients,
-Enables MDBs to listen for changes for the files they are interested in. Such file state changes are abstracted as "messages" and delivered to the interested MDBs.
-Fulfills other expectations from transactional system : deadlock detection, concurrency control, crash-recovery, transaction timing-out.
-Supports both local and XA transactions.
____________________________________________

Here is the current status from testing:

-has been tested against some tests of simple to medium complexity.
-i feel a need to test it further with more and more rigorous tests, because being a transactional system, the software will be handed-over critical data and so need to create as much trust as it can."
