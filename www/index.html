XADisk provides transactional access to file-systems from Java and JavaEE applications.

 

Features

  • Supports both XA and plain local transactions.
  • Is JCA 1.5 compliant, so can plug into any JavaEE 1.5+ Server. JavaEE applications can interact with XADisk it as a JCA Resource Adapter.
  • Standalone Java applications can also be clients.
  • Enables MDBs to listen for changes for the files they are interested in.
  • Fulfills other expectations from transactional system : deadlock detection, concurrency control, crash-recovery, transaction timing-out etc.
  • Can be invoked from Java/JavaEE applications running on remote JVMs/machines.
